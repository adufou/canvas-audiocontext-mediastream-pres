1

AudioContext + OscillatorNode

Browsers require a user gesture before any audio can play. Click Start AudioContext first — this unlocks the audio system. OscillatorNodes are one-shot: a new node is created on each Play click.

not started
2

GainNode — volume control

A GainNode multiplies the signal by its gain.value. Audio parameters can be set instantly or automated over time with .setValueAtTime(), .linearRampToValueAtTime(), etc.

3

BiquadFilterNode — EQ / filtering

Filters shape the frequency spectrum. Try a lowpass at 500 Hz with the square wave — you'll hear the harmonics disappear, rounding the harsh tone.

4

AnalyserNode → Canvas — real-time visualizer

The AnalyserNode exposes the audio signal as raw data — either time-domain (waveform) or frequency-domain (FFT spectrum). We draw it on a Canvas each frame with requestAnimationFrame.

5

Full signal chain

Wire all nodes with chained .connect() calls. Watch the signal flow diagram above light up as the audio passes through each node.

audiocontext.html JS 
// Step 1 — AudioContext + OscillatorNode

// Must be created inside a user-gesture handler
const audioCtx = new AudioContext();
await audioCtx.resume();   // ensure it's running

// OscillatorNode generates a tone
// IMPORTANT: oscillators are one-shot.
// You can only call .start() once per node.
// Create a new oscillator each time you want to play.
let osc = audioCtx.createOscillator();
osc.type      = 'sine';        // sine | square | triangle | sawtooth
osc.frequency.setValueAtTime(440, audioCtx.currentTime);

osc.connect(audioCtx.destination);
osc.start();

// To stop:
osc.stop();
osc.disconnect();
// Step 2 — GainNode

const gain = audioCtx.createGain();
gain.gain.value = 0.7;  // 0 = silence, 1 = full volume

// Wire: oscillator → gain → speakers
osc.connect(gain);
gain.connect(audioCtx.destination);

// Update in real time (no clicks/pops at high gain values)
gain.gain.setTargetAtTime(
  newVolume,
  audioCtx.currentTime,
  0.01          // time constant (seconds)
);

// Or simply for non-critical changes:
gain.gain.value = 0.5;
// Step 3 — BiquadFilterNode

const filter = audioCtx.createBiquadFilter();

filter.type            = 'lowpass'; // lowpass | highpass | bandpass...
filter.frequency.value = 2000;    // cutoff frequency in Hz
filter.Q.value         = 1;        // resonance (bandwidth shape)

// Wire: oscillator → gain → filter → speakers
osc.connect(gain);
gain.connect(filter);
filter.connect(audioCtx.destination);

// Change in real time — just update the values
filter.frequency.value = 500;
filter.type            = 'highpass';
// Step 4 — AnalyserNode + Canvas visualizer

const analyser = audioCtx.createAnalyser();
analyser.fftSize = 2048;  // must be power of 2

// Wire analyser into the chain (pass-through)
filter.connect(analyser);
analyser.connect(audioCtx.destination);

// Frequency-domain (FFT spectrum)
const freqData = new Uint8Array(analyser.frequencyBinCount);
analyser.getByteFrequencyData(freqData);

// Time-domain (waveform)
const timeData = new Uint8Array(analyser.fftSize);
analyser.getByteTimeDomainData(timeData);

// Draw loop — runs until we set animating = false
let animating = true;
function drawLoop() {
  if (!animating) return;
  requestAnimationFrame(drawLoop);
  analyser.getByteFrequencyData(freqData);
  // ... draw bars on canvas ...
}
// Step 5 — Full signal chain

// .connect() returns the destination node,
// so you can chain calls:
osc
  .connect(gain)
  .connect(filter)
  .connect(analyser)
  .connect(audioCtx.destination);

osc.start();

// To stop cleanly:
osc.stop();        // fires osc.onended

osc.onended = () => {
  osc.disconnect();
  // stop the visualizer draw loop
  animating = false;
};
